# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.1.x   | :white_check_mark: |
| < 1.1   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in Sitefinity CLI, please report it responsibly:

1. **Do not** open a public GitHub issue for security vulnerabilities.
2. Email the maintainers or use [GitHub Security Advisories](https://github.com/Sitefinity/Sitefinity-CLI/security/advisories/new) to report the issue privately.
3. Include a description of the vulnerability, steps to reproduce, and potential impact.

We will acknowledge receipt within 3 business days and aim to provide a fix or mitigation within 30 days.

## Security Best Practices

- Keep your Sitefinity CLI installation up to date.
- Review dependency updates provided by Dependabot pull requests.
- Use `dotnet restore` with NuGet audit enabled to detect vulnerable packages.
